# Candidate B

## Answer

As checked on 2026-07-13, RFC 9110, **HTTP Semantics**, is an **Internet Standard** and is assigned **STD 97**. **June 2022 is its publication date**; the document header's `Category: Standards Track` describes its standards-track category and should not be substituted for the current status label shown by the IETF Datatracker and RFC Editor.

Its intended role is the version-independent semantic core of HTTP: it defines the overall architecture, common terminology, core protocol elements, extensibility mechanisms, the `http` and `https` URI schemes, and semantics shared by HTTP versions. It updates RFC 3864 and obsoletes the listed older semantics-related RFCs and part of RFC 7230. It should therefore be used for HTTP meaning and behavior, alongside version-specific specifications for wire messaging rather than as a replacement for every HTTP specification.

## Boundaries

This conclusion is bounded to the current official IETF Datatracker record and RFC Editor document record. Status, errata, or replacement relationships can change; recheck the RFC Editor information page when a coding decision depends on current status. This review does **not** infer that no successor exists merely because none appeared in the inspected records.
